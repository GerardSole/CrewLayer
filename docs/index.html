Introduction

Getting Started · 5 min read

CrewLayer is an open-source backend for AI agents. It provides persistent memory, shared context, a full action history, and multi-tenant isolation — all through a simple REST API and official SDKs.

The goal is that any developer can wire their agent into CrewLayer in minutes and never worry about state management, memory, or logs again. It works with any agent framework: LangChain, AutoGen, LlamaIndex, Vercel AI SDK, CrewAI, or plain Python.

CrewLayer is self-hosted, MIT-licensed, and ships as a single docker compose up -d command. No SaaS pricing. No data leaving your infrastructure. You own everything.

Architecture

CrewLayer is a FastAPI application backed by PostgreSQL (with pgvector) for long-term storage and Redis for short-term memory and pub/sub:

What it gives you

  • Persistent Memory — Short-term messages in Redis per session, long-term facts in pgvector with semantic recall.
  • Action Logger — Immutable append-only log of every tool call. Full input/output and timing.
  • Shared Blackboard — Multi-agent coordination via a namespaced key-value store with Redis pub/sub and history.
  • Sessions & Episodes — Group messages into sessions, sessions into episodes. Extract memories when sessions close.
  • Evaluations & A/B Testing — LLM-as-a-judge scoring, anomaly detection, prompt versioning with rollback.
  • Multi-tenant — Every resource is isolated per tenant. API keys with granular scopes.
  • MCP Server — Use all features from Claude Desktop or Claude Code without writing HTTP.
  • Web Dashboard — Built-in React dashboard at /dashboard. Memory explorer, action timeline, A/B results, agent status.
  • Observability — Prometheus metrics at /metrics and a prebuilt Grafana dashboard.
